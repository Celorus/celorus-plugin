# Composition Contract — the anatomy gate

Tokens have had hard gates from day one (contrast build, hex policy). Composition — the
thing that actually makes an artifact read as Celorus — had none, and the 2026-07-08 audit
showed the consequence: a token-perfect artifact that missed the anatomy read as a pale
cousin, while an off-token page that nailed the anatomy read as the brand. This contract
makes the anatomy checkable. `tools/composition-lint.mjs` is the executable form; this file
is the reasoning and the rules the lint cannot check.

## Declare the surface

Every HTML artifact declares what it is, in `<head>`:

```html
<meta name="celorus-surface" content="collateral">
```

| class | means | start from |
|---|---|---|
| `collateral` | standalone data/report page (report, cap table, FH, synthesis, form) | `templates/html/report.html` etc. |
| `command` | command view — ops command, control center, owner brief | `templates/html/ops-command.html` |
| `tier1-artifact` | full-page hosted artifact (claude.ai artifact viewer) | same anatomy as collateral/command + embedded fonts |
| `tier2-embedded` | plugin panel, iframe, chat widget, ≤480px card | `guides/PLUGIN_SURFACES.md` |
| `app` | app-shell UI (sidebar/topbar product surface) | `templates/html/dashboard.html` |
| `deck` | slide deck | `templates/html/deck.html` |
| `marketing` | website page, one-pager | `marketing/` |
| `banner` | fixed-size export surface | `marketing/banners/` |
| `email` | the one sanctioned inline-hex surface | `marketing/email/` |

An undeclared surface is a lint ERROR. The declaration is the contract handle: it tells
the lint — and the next maker — which anatomy this page owes.

## What the lint enforces (per class)

Errors fail CI; warnings print and pass. See the matrix in `tools/composition-lint.mjs`
(it is the source of truth for levels).

- **Theme bridge** — `prefers-color-scheme` detection + `data-theme` stamp, print always
  light. All classes except banner/email.
- **Provenance** — "Generated by Celorus · date · sources cited inline". Everything except
  banner/email; the live marketing site is a documented warn-level exemption (AGENTS rule 4).
- **Charcoal topbar** — required on collateral / command / tier1-artifact; **forbidden
  (warn) on tier2-embedded** — the host owns the window there.
- **Eyebrow → h2 rhythm** — warn-level heuristic; charcoal close bands are exempt (they
  open with a plain serif h2 by design).
- **Hex literals** — scanned in CSS scope only (`<style>` + `style=""`), data-URIs
  excluded, so prose issue-refs can't false-positive. The inlined GENERATED foundation
  paste-block is sanctioned (SKILL steps 2/5 mandate inlining it where CSP blocks
  `<link>`; its hexes are the tokens, gated where they are built): the scan skips from
  the block's header sentinel to the end of that `<style>` block, and skips explicit
  `/* lint:allow-token-block */ … /* lint:end-token-block */` fences — closed pairs
  only, an unclosed fence sanctions nothing. Everything else stays strict
  (`tools/composition-lint.test.mjs` pins both directions).
- **Type floor** — literal `font-size` below 9px is an error, 9–10px a warning. The token
  floor is 10px (`fontSize-3xs`); micro-type creep is how dashboards rot.
- **Embedded fonts** — tier1-artifact must carry `data:font/woff2` (CSP-sandboxed hosts
  block font CDNs; inline `tokens/dist/embedded-fonts.css`).

## What the lint cannot check — still binding

1. **Curate, don't index.** A command view keeps ≤10 items in the priorities rail above
   the fold; everything else collapses into its domain panel. An exhaustive index is a
   different artifact from a briefing — if you need both, the briefing layer goes on top.
2. **Verdict headlines.** The hero states a verdict ("What's running, and what needs you"),
   not a topic ("Status dashboard"). At most one phrase recolored petrol via `<em>`.
3. **Stat values render serif at display scale** (26px+), never mono-small. Mono is for
   identifiers and data cells, not for the number the reader came for.
4. **Word + color, never color alone** — status pills carry the word, dots pair with text.
5. **Blind spots are declared** — a "?" value ships with an enable action, never as an
   unexplained gap.

## Running it

```bash
node tools/composition-lint.mjs templates/html/*.html marketing/landing.html \
  marketing/one-pager.html marketing/banners/*.html marketing/email/email.html
```

CI runs exactly this (`.github/workflows/ci.yml`) plus the token/contrast gate on every
push and PR. For a new artifact outside the repo, run the lint on the file before
delivering; the celorus-design skill bundles a copy (`assets/composition-lint.mjs`).
